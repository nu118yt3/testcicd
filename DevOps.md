# Guía de Estudio Exhaustiva DevOps/Cloud Native  
Estructurada por Tecnología (Orientada a Entrevistas Senior/Arquitecto)

| **1. Kubernetes**   | Orquestador de contenedores de código abierto líder. Automatiza el despliegue, escalado y operación de aplicaciones contenerizadas en clústeres. Es el estándar de facto en entornos Cloud Native (CNCF graduated project). | • Componentes del clúster (Control Plane: API Server, etcd, Scheduler, Controller Manager, cloud-controller-manager; Nodos: kubelet, kube-proxy, container runtime)<br>• Objetos primitivos y workloads (Pod, ReplicaSet, Deployment, StatefulSet, DaemonSet, Job/CronJob)<br>• Namespaces, Labels & Selectors, Annotations<br>• CNI (Calico, Cilium, Flannel), CSI, StorageClass, PV/PVC<br>• Helm v3 (Charts, Release, Repository, ChartMuseum/Helm Hub)<br>• kubectl (imperative vs declarative, dry-run, context/config) | • Describa en detalle los componentes del Control Plane y qué ocurre si falla etcd.<br>• ¿Cómo desplegaría una aplicación stateful (ej. MySQL/Wordpress) en producción? → StatefulSet + PVC + Headless Service + anti-affinity.<br>• Explique el ciclo de vida de un Pod y los estados (Pending → Running → Succeeded/Failed).<br>• Un Pod se queda en CrashLoopBackOff → ¿qué comandos ejecuta en orden? (`kubectl logs`, `events`, `describe pod`, `exec`).<br>• ¿Cómo instala y mantiene Helm v3 en un equipo? ¿Qué cambió respecto a Helm v2 (Tiller)?<br>• Escenario EKS: ¿Cómo asegura alta disponibilidad de Jenkins en Kubernetes usando Helm + Portworx/EBS CSI? |
| **2. Docker**       | Plataforma de contenerización que permite empaquetar aplicaciones con todas sus dependencias en imágenes ligeras y portables. Base fundamental de todo el ecosistema Cloud Native.                                          | • Docker Engine (daemon dockerd + CLI)<br>• Imágenes vs Contenedores<br>• Capas (layer caching), Dockerfile best practices<br>• Docker Compose v2 (docker-compose.yml)<br>• Multi-stage builds<br>• Docker Hub / registries privados (Harbor, ECR, ACR)<br>• Hardened/Slim images (Distroless, UBI, Red Hat Universal Base Image)<br>• cAdvisor para métricas | • Diferencia entre Docker Engine y Docker Compose. ¿Cuándo usa uno u otro en un pipeline?<br>• ¿Cómo integra Docker en un pipeline CI/CD de Jenkins? → Uso del plugin Docker Pipeline, agentes docker, `docker.build`, `docker.push`, `docker.run`, `docker inside`.<br>• ¿Cómo reduce el tamaño y los CVEs de sus imágenes en producción? → Multi-stage, base images hardened (Distroless), scanning con Trivy/Clair.<br>• ¿Cómo monitorea contenedores Docker con Prometheus? → cAdvisor expone métricas en /metrics, se añade como job en prometheus.yml.<br>• Explique `docker system prune` y riesgos en producción. |
| **3. Ansible**      | Herramienta de automatización agentless (sin agentes) basada en YAML que permite gestión de configuración, despliegue y orquestación a gran escala. Muy utilizada junto a Terraform.                                            | • Playbooks, Tasks, Modules, Roles, Collections<br>• Inventario (estático/dinámico)<br>• Arquitectura agentless (SSH + Python)<br>• Precedencia de variables (22 niveles → más alta: `--extra-vars` CLI)<br>• Idempotencia<br>• Molecule para testing de roles<br>• Ansible Tower / AWX (enterprise) | • ¿Cuál es la diferencia práctica entre Ansible y Terraform? → Terraform = IaC declarativo para infraestructura; Ansible = gestión de configuración y orquestación (imperativo/declarativo).<br>• Explique la precedencia de variables y dé el orden de los 3 niveles más altos.<br>• ¿Cómo prueba un role antes de ejecutarlo en producción? → Molecule + Docker/Vagrant/Podman como driver.<br>• ¿Cómo escala Ansible a 10 000+ nodos? → Serial/batching, strategy: free, forks, pipelining, AWX/Tower queues.<br>• Escenario: Un playbook falla por “variable undefined” → ¿cómo lo debuggea? (`--check --diff`, `ansible-playbook --step`, `debug: var=`). |
| **4. Jenkins CI/CD**| Servidor de automatización open-source líder para construir, probar y desplegar código mediante pipelines como código (Jenkinsfile). Es el orquestador CI/CD más usado en empresas.                                              | • Arquitectura Master-Agent (Controller + Agents)<br>• Pipeline Declarativo vs Scripted<br>• Jenkinsfile (pipeline {}, stages, steps, post)<br>• Shared Libraries<br>• Jenkins Configuration as Code (JCasC)<br>• Credentials Plugin (secrets management)<br>• Blue Ocean UI<br>• Alta disponibilidad en Kubernetes | • ¿Declarative o Scripted Pipeline? ¿Cuándo y por qué prefiere uno? → Declarative 95 % casos (legible, restartable); Scripted solo para lógica muy compleja.<br>• ¿Cómo protege secretos en un pipeline? → Credentials Binding Plugin + `withCredentials`.<br>• ¿Cómo despliega Jenkins HA en producción? → Helm chart oficial en EKS + PersistentVolume (EFS o Portworx) + Ingress/TLS + autoscaling de agentes con Kubernetes Plugin.<br>• Escenario: El agente se desconecta constantemente → ¿qué revisa? (Java version, JNLP port, logs agente, tunnel, `-direct` connection).<br>• Explique `node` vs `stage` vs `step`. |
| **5. Prometheus**   | Sistema de monitoreo y alerting basado en series temporales diseñado para cloud-native. Junto con Grafana y Alertmanager forma el stack de observabilidad estándar en Kubernetes.                                               | • Pull model (scraping)<br>• Modelo de datos dimensional (metric name + labels)<br>• PromQL (instant vs range vectors, functions, operators)<br>• Service Discovery (kube sd_config, consul, etc.)<br>• Exporters (Node, Blackbox, JMX, MySQL, etc.)<br>• Alertmanager (routing, inhibition, silencing)<br>• Thanos / Cortex / Mimir para HA y long-term storage | • Describa el ecosistema completo de Prometheus y cómo logra HA/escalabilidad → Thanos (Sidecar, Querier, Store, Compactor, Ruler).<br>• Escenario: “context deadline exceeded” en un target → ¿qué pasos sigue? (curl al /metrics, aumentar scrape_timeout, revisar firewall, revisar relabel_configs, promtool check config).<br>• ¿Cómo crea una alerta de “Pod en CrashLoopBackOff” con PromQL?<br>• Explique recording rules y alerting rules, ¿cuándo usar cada una?<br>• ¿Cómo federar métricas entre clústeres diferentes? → Thanos Querier + federation scrape. |

### Consejos Generales para la Entrevista
- Siempre menciona herramientas enterprise cuando corresponda: Helm, Harbor, Portworx, AWX/Tower, Jenkins HA en EKS, Thanos.
- Relaciona todo con pipelines reales y escalabilidad (10k+ nodos, clústeres multi-AZ).
- Usa comandos concretos (`kubectl`, `ansible-playbook`, `promtool`, `helm`) → demuestra experiencia práctica.
- Seguridad es clave: hardened images, RBAC, secrets management, TLS everywhere.
